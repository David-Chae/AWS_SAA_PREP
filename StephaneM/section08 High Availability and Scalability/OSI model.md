
**OSI 7계층(Open Systems Interconnection Model)**은
컴퓨터가 **네트워크를 통해 데이터를 주고받는 과정을 단계별로 나눈 모델**이에요.

쉽게 말하면,
👉 “사람이 편지를 보내는 과정”을 **7단계로 나눈 설명서** 같은 거예요.

---

## 💌 OSI 7계층을 ‘편지 보내기’로 비유해보기

| 계층      | 이름                    | 하는 일                     | 편지 비유                     |
| ------- | --------------------- | ------------------------ | ------------------------- |
| **7계층** | 응용 계층 (Application)   | 사용자가 직접 보는 부분 — 웹, 이메일 등 | “편지 내용을 쓰는 사람”            |
| **6계층** | 표현 계층 (Presentation)  | 데이터의 형식 변환, 암호화/복호화      | “언어 번역, 글씨 정리”            |
| **5계층** | 세션 계층 (Session)       | 연결 관리 — 언제 통신 시작하고 끊을지   | “상대방과 통화 연결 및 종료”         |
| **4계층** | 전송 계층 (Transport)     | 데이터가 잘 전달되도록 조각내고 재조합    | “편지를 봉투 여러 개에 나눠서 정확히 배달” |
| **3계층** | 네트워크 계층 (Network)     | 경로 선택, IP 주소 기반 전송       | “어떤 길(우편 경로)로 보낼지 결정”     |
| **2계층** | 데이터 링크 계층 (Data Link) | 물리적인 주소(MAC 주소)로 전송      | “우체국 직원이 주소 라벨을 붙임”       |
| **1계층** | 물리 계층 (Physical)      | 전기 신호, 케이블, 와이파이 등 실제 전달 | “우체부가 자전거 타고 실제로 배달”      |

### 🪄 전체 비유 정리

📱 **Application (7–5층)** — 사람이 “말하고 듣는” 단계
🚚 **Transport (4–3층)** — 데이터가 “이동하는” 단계
🔌 **Network Access (2–1층)** — 실제 “배달 수단”

---

즉,

* **1~3계층**: 네트워크가 데이터를 실제로 전달
* **4계층**: 데이터를 안전하게 전달 (TCP/UDP)
* **5~7계층**: 사용자에게 보이도록 서비스 형태로 바꿈
