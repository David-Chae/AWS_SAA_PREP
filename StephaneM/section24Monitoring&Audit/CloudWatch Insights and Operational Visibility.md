## CloudWatch Insights Overview  
## CloudWatch Insights 개요  
Let's discuss the insights types of CloudWatch products. The first one is CloudWatch Container Insights.  
CloudWatch 제품의 인사이트 유형을 살펴보겠습니다. 첫 번째는 CloudWatch Container Insights입니다.  
(CloudWatch 인사이트 소개)  

CloudWatch Container Insights collects, aggregates, and summarizes metrics and logs from your containers.  
CloudWatch Container Insights는 컨테이너의 메트릭과 로그를 수집, 집계, 요약합니다.  
(컨테이너 모니터링 기능)  

It is available for containers running on Amazon ECS, Amazon EKS, Kubernetes directly on EC2, or on Fargate for both ECS and EKS.  
이 기능은 Amazon ECS, Amazon EKS, EC2에서 직접 실행되는 Kubernetes, ECS 및 EKS용 Fargate 컨테이너에서 사용할 수 있습니다.  
(지원 플랫폼 설명)  

Thanks to CloudWatch Container Insights, you can easily extract metrics and logs from your containers into a detailed and granular dashboard within CloudWatch.  
CloudWatch Container Insights 덕분에 컨테이너의 메트릭과 로그를 CloudWatch 내 세부 대시보드로 쉽게 추출할 수 있습니다.  
(대시보드 활용 설명)  

If you are using CloudWatch Container Insights on Kubernetes, whether on Amazon EKS or Kubernetes running on EC2, it uses a containerized version of the CloudWatch agents to discover containers.  
Kubernetes에서 CloudWatch Container Insights를 사용하는 경우, Amazon EKS든 EC2에서 실행되는 Kubernetes든 CloudWatch 에이전트의 컨테이너화된 버전을 사용하여 컨테이너를 발견합니다.  
(에이전트 동작 설명)  

---

## Lambda Insights  
## Lambda Insights  
Lambda Insights is a monitoring and troubleshooting solution for your serverless applications running on AWS Lambda.  
Lambda Insights는 AWS Lambda에서 실행되는 서버리스 애플리케이션의 모니터링 및 문제 해결 솔루션입니다.  
(서버리스 모니터링)  

It collects, aggregates, and summarizes system-level metrics including CPU time, memory, disk, network, cold starts, and Lambda worker shutdowns.  
CPU 시간, 메모리, 디스크, 네트워크, 콜드 스타트, Lambda 워커 종료 등 시스템 수준 메트릭을 수집, 집계, 요약합니다.  
(수집 메트릭 설명)  

Lambda Insights is provided as a Lambda layer that runs alongside your Lambda function.  
Lambda Insights는 Lambda 함수와 함께 실행되는 Lambda 레이어로 제공됩니다.  
(레이어 형태 설명)  

It creates a specified dashboard called Lambda Insights to monitor the performance of your Lambda functions.  
Lambda 함수 성능을 모니터링하기 위해 Lambda Insights라는 지정 대시보드를 생성합니다.  
(대시보드 설명)  

This is useful when you need detailed monitoring of your serverless applications.  
서버리스 애플리케이션의 세부 모니터링이 필요할 때 유용합니다.  
(활용 이유)  

---

## Contributor Insights  
## Contributor Insights  
Contributor Insights analyzes logs and creates time series that display contributed data.  
Contributor Insights는 로그를 분석하고 기여 데이터를 표시하는 시계열을 생성합니다.  
(로그 분석 기능)  

For example, it helps identify the top contributors or the total number of unique contributors and their usage.  
예를 들어, 주요 기여자나 고유 기여자의 총 수와 사용량을 식별하는 데 도움을 줍니다.  
(활용 사례)  

This helps find top talkers on your network and understand who is impacting system performance.  
네트워크 상의 주요 사용자를 찾아 시스템 성능에 영향을 주는 주체를 이해하는 데 도움을 줍니다.  
(분석 목적)  

It can run on any logs generated by AWS, such as VPC logs or DNS logs.  
VPC 로그나 DNS 로그 등 AWS에서 생성된 모든 로그에서 실행될 수 있습니다.  
(지원 로그 유형)  

For example, by analyzing VPC flow logs, which record all network requests within your VPC and are passed to CloudWatch Logs, Contributor Insights can identify the top 10 IP addresses generating traffic.  
예를 들어, VPC 플로우 로그를 분석하여 트래픽을 생성하는 상위 10개 IP 주소를 식별할 수 있습니다.  
(실무 예시)  

This helps determine if they are good or bad actors.  
이 정보를 통해 해당 IP가 정상 사용자인지 악성 사용자인지 판단할 수 있습니다.  
(활용 목적)  

You can build rules from scratch or leverage simple rules that AWS has created.  
규칙을 처음부터 만들거나 AWS에서 제공하는 간단한 규칙을 활용할 수 있습니다.  
(규칙 구성)  

Behind the scenes, Contributor Insights leverages CloudWatch Logs and includes built-in rules to analyze metrics from other AWS services.  
Contributor Insights는 내부적으로 CloudWatch Logs를 활용하며, 다른 AWS 서비스 메트릭 분석용 내장 규칙을 포함합니다.  
(동작 원리)

## CloudWatch Application Insights  
## CloudWatch Application Insights  
CloudWatch Application Insights provides an automated dashboard that shows potential problems with monitored applications to help isolate ongoing issues.  
CloudWatch Application Insights는 모니터링되는 애플리케이션의 잠재적 문제를 표시하는 자동 대시보드를 제공하여 문제를 빠르게 분리할 수 있게 합니다.  
(자동 대시보드 기능)  

Your applications can be running on Amazon EC2.  
애플리케이션은 Amazon EC2에서 실행될 수 있습니다.  
(지원 환경)  

You select technologies such as Java, .NET, Microsoft IIS web server, or specific databases.  
Java, .NET, Microsoft IIS 웹 서버 또는 특정 데이터베이스와 같은 기술을 선택할 수 있습니다.  
(지원 기술)  

It links to other AWS resources like EBS, RDS, ELB, ASG, Lambda, SQS, DynamoDB, S3 buckets, ECS clusters, EKS clusters, SNS topics, or API Gateway.  
EBS, RDS, ELB, ASG, Lambda, SQS, DynamoDB, S3 버킷, ECS/EKS 클러스터, SNS 토픽, API Gateway 등 다른 AWS 리소스와 연결됩니다.  
(리소스 연동)  

If there is an issue with your application, CloudWatch Application Insights automatically creates a dashboard showing the potential issue with some services.  
애플리케이션에 문제가 발생하면 CloudWatch Application Insights는 일부 서비스와 관련된 잠재적 문제를 보여주는 대시보드를 자동으로 생성합니다.  
(자동 문제 표시)  

It uses SageMaker machine learning internally to enhance visibility into your application health, reducing troubleshooting and repair time.  
내부적으로 SageMaker 머신러닝을 사용하여 애플리케이션 상태 가시성을 높여 문제 해결과 수리 시간을 줄입니다.  
(ML 활용)  

All alerts and findings are sent to Amazon EventBridge and SSM OpsCenter so you can be alerted if an issue is detected with your application.  
모든 경고와 결과는 Amazon EventBridge 및 SSM OpsCenter로 전송되어 애플리케이션 문제 감지 시 알림을 받을 수 있습니다.  
(알림 전달)

## Summary  
## 요약  
To summarize, CloudWatch Container Insights is for metrics and logs from ECS, EKS, Kubernetes on EC2, and Fargate. For Kubernetes, an agent is required to run.  
정리하면, CloudWatch Container Insights는 ECS, EKS, EC2의 Kubernetes, Fargate 컨테이너의 메트릭과 로그를 제공합니다. Kubernetes는 에이전트 실행이 필요합니다.  
(서비스 요약)  

CloudWatch Lambda Insights provides detailed metrics to troubleshoot serverless applications running on AWS Lambda.  
CloudWatch Lambda Insights는 AWS Lambda에서 실행되는 서버리스 애플리케이션 문제 해결을 위한 세부 메트릭을 제공합니다.  
(서비스 요약)  

Contributor Insights helps find top contributors through CloudWatch Logs.  
Contributor Insights는 CloudWatch Logs를 통해 주요 기여자를 식별하는 데 도움을 줍니다.  
(서비스 요약)  

Application Insights creates automatic dashboards to troubleshoot applications and related AWS services.  
Application Insights는 애플리케이션과 관련 AWS 서비스 문제 해결을 위해 자동 대시보드를 생성합니다.  
(서비스 요약)

## Key Takeaways  
## 핵심 요약  
- CloudWatch Container Insights collects, aggregates, and summarizes metrics and logs from containers running on ECS, EKS, Kubernetes on EC2, and Fargate.  
- CloudWatch Container Insights는 ECS, EKS, EC2의 Kubernetes, Fargate에서 실행되는 컨테이너의 메트릭과 로그를 수집, 집계, 요약합니다.  

- Lambda Insights provides detailed monitoring and troubleshooting for AWS Lambda functions, including metrics like CPU, memory, disk, network, cold starts, and worker shutdowns.  
- Lambda Insights는 CPU, 메모리, 디스크, 네트워크, 콜드 스타트, 워커 종료 등 메트릭을 포함하여 AWS Lambda 함수의 세부 모니터링과 문제 해결을 제공합니다.  

- Contributor Insights analyzes logs to identify top contributors impacting system performance, such as heavy network users or error-generating URLs.  
- Contributor Insights는 로그를 분석하여 시스템 성능에 영향을 주는 주요 기여자를 식별합니다. (예: 네트워크 과다 사용자, 에러 URL)

* CloudWatch Application Insights offers automated dashboards using machine learning to detect potential issues in applications and related AWS resources, facilitating faster troubleshooting.
* CloudWatch Application Insights는 머신러닝 기반 자동 대시보드를 제공하여 애플리케이션 및 관련 AWS 리소스의 잠재적 문제를 감지하고 빠른 문제 해결을 지원합니다.
