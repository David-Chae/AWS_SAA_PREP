# ⭐ Amazon ElastiCache 쉽게 이해하기

## 1. **ElastiCache란?**

AWS가 제공하는 **Redis / Memcached 캐시 서버를 대신 관리해주는 서비스**야.
RDS가 데이터베이스를 관리형으로 제공하는 것처럼,
ElastiCache는 **캐시 시스템을 관리형으로 제공**한다고 보면 돼.

---

# ⚡ 2. 캐시(Cache)란 무엇인가?

캐시는 **메모리(RAM) 안에 데이터를 저장해서 엄청 빠르게 가져오는 저장소**야.

* 일반 DB: 밀리초 단위 (상대적으로 느림)
* 캐시: **서브 밀리초(거의 즉시)**

즉, “자주 요청되는 데이터”를 캐시에 넣어두면
서비스 속도가 훨씬 빨라지고 DB 부담이 줄어.

---

# 🔧 3. Redis를 쓸 때 가능한 기능들

ElastiCache for Redis를 사용하면 다음 기능을 사용할 수 있어:

* **클러스터링**(데이터를 여러 노드로 나눠 저장)
* **Multi-AZ** (장애 대비용 이중화)
* **Read Replicas** (읽기 성능 확장)
* **인증, 암호화(KMS), 보안그룹 등** 보안 통합
* **백업, 스냅샷, 포인트 인 타임 복구** 가능

즉, Redis 서버 관리와 설정을 AWS가 대신 해주는 셈.

---

# ❗ 4. 가장 중요한 포인트 (시험에도 자주 나옴)

**ElastiCache를 쓰면 ‘반드시’ 애플리케이션 코드를 수정해야 한다.**

예)

* DB 조회 → 캐시 조회 후 없으면 DB 조회 → 캐시에 저장
* 세션 저장 로직을 Redis로 변경
* 자주 쓰는 데이터만 Redis에 넣도록 코드 수정

즉, **코드 변경 없이 캐싱을 쓰고 싶다 → ElastiCache 선택 X**
(이건 exam에서 자주 나오는 트릭)

---

# 🎯 5. 어디에 사용하나? (주로 쓰는 대표 상황)

### ✔ 자주 읽히는 데이터 캐싱

DB 부하를 줄이고 조회 속도를 빠르게 하고 싶을 때.

### ✔ 세션 저장

웹사이트 로그인 세션을 Redis에 저장하면
서버 여러 대가 있어도 같은 세션을 공유할 수 있어.

### ✔ 단순 Key-Value 저장소

예:

* “user:123 → userObject”
* “product:998 → productDetail”

단, **SQL은 사용할 수 없음**.
캐시는 SQL DB가 아니라 단순 Key/Value 저장공간이야.
