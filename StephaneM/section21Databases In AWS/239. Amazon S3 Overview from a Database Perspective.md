
# ⭐ Amazon S3를 데이터베이스 관점에서 이해하기

## 1. **S3는 ‘대형 창고형 Key-Value Store’다**

* **Key = 파일 경로(객체 이름)**
* **Value = 실제 파일(객체)**
* 파일 하나에 최대 **5TB까지 저장 가능**
* **엄청 큰 파일 저장에 최적화** (영상, 로그 덩어리, 백업 등)
* 아주 작은 파일을 수백만 개 저장할 때는 효율이 떨어질 수 있음
* 서버 관리 필요 없음 → **무한히 자동 확장**

---

# 🧊 2. 저장 계층(티어) = “냉장고/냉동고처럼 비용 최적화”

S3는 데이터의 ‘온도’에 맞게 여러 티어를 제공함:

* **S3 Standard:** 자주 읽는 뜨거운 파일
* **Infrequent Access:** 가끔 보는 파일
* **Intelligent Tiering:** 사용 패턴 보면서 자동 이동
* **Glacier:** 거의 안 보는 장기 보관(냉동 창고)

→ **Lifecycle 정책**으로 자동 이동 설정 가능
예: 30일 지나면 Glacier로 보내기

---

# 📦 3. 핵심 기능

### ✔ 버전관리 (Versioning)

파일의 이전 버전까지 보관 → 실수로 덮어써도 복구 가능

### ✔ 암호화

서버 측, KMS 키 기반, 고객 제공 키 등 다양한 방식 가능

### ✔ 복제

버킷끼리 자동 복사(지역 간 복제도 가능)

### ✔ MFA Delete

삭제하려면 MFA 필요 → 오남용/오삭제 방지

### ✔ 액세스 로그

누가 접근했는지 남길 수 있음

---

# 🔐 4. 보안 방식

S3 보안은 여러 층으로 이루어짐:

* **IAM 정책** → 어떤 사람/서비스가 어떤 버킷을 조작할 수 있는가
* **Bucket Policy** → 버킷 자체에 걸리는 규칙
* **ACL** → 파일 단위 옛날식 권한
* **Access Point** → 대규모 공유 환경에서 편하게 권한 제어
* **MFA Delete** → 삭제 보호

결론: **접근 통제는 IAM + 버킷 정책이 핵심**

---

# 🧩 5. 고급 기능

### ✔ S3 Object Lambda

파일을 다운로드하기 전에 **가공해서 내려보낼 수 있음**
예: 이미지 원본 대신 썸네일 생성 후 전달

### ✔ CORS

웹 브라우저에서 다른 도메인의 S3 파일 접근 허용 여부 설정

### ✔ Object Lock / Glacier Vault Lock

규정 준수, 변경 불가 상태 유지 (WORM 규칙)

---

# 🛡️ 6. 암호화 총정리

S3는 여러 방식으로 암호화를 지원해:

* **SSE-S3:** S3가 키 관리
* **SSE-KMS:** KMS 키 사용(더 안전, 로그 가능)
* **SSE-C:** 고객이 직접 키 제공
* **Client-side encryption:** 업로드 전에 직접 암호화
* **전송 중:** TLS로 보호

버킷에 **기본 암호화 설정**도 가능함.

---

# 📊 7. 대량 작업 지원

### ✔ S3 Batch Operations

수백만 개 객체에 대해 한 번에 작업
예: 암호화 안 된 객체 모두 암호화하기
예: 버킷 A → B로 대량 복사하기

### ✔ S3 Inventory

“버킷 안에 어떤 객체가 있는지 목록 생성”
→ Batch Operations에 주로 사용됨

---

# ⚡ 8. 성능 기능

### ✔ 멀티파트 업로드

큰 파일을 여러 조각으로 나눠 병렬 업로드 → 훨씬 빠름

### ✔ Transfer Acceleration

멀리 떨어진 지역에서 S3 업로드 속도 가속

### ✔ S3 Select

파일 전체를 받지 않고,
**CSV/JSON/Parquet 안에서 필요한 부분만 선택해서 읽기**

→ 비용/속도 모두 절약

---

# 🔔 9. 자동 이벤트 처리

S3에 어떤 일이 발생하면 자동으로 알림/처리 가능:

* 파일 업로드됨 → Lambda 자동 실행
* 파일 삭제됨 → SNS 알림
* 새 파일 생성 → SQS Queue에 메시지 전송
* EventBridge로 규칙 기반 처리

→ S3를 **이벤트 중심 아키텍처의 시작점으로 사용** 가능

---

# 🧰 10. 언제 S3를 쓰는가?

* 사진, 영상, 로그, 문서 등 **대용량 파일 저장**
* 정적 웹사이트 호스팅
* 백업/아카이브 저장
* 데이터 레이크 만들기 (Athena와 함께)
* 분석용 원시 데이터 저장소
* 여러 서비스의 파일 중앙 저장소
